
# DosePair Build Context

```

  index.js.v1         Main server entry point

  validate.js.v1      Input validation
  security-test.js.v1 Security test suite (jest + supertest)

  actions/  display/  feedback/  forms/  navigation/

assets/               logo, fonts
ui_kits/
  dosepair_web/       5-screen web app recreation
  dosepair_ios/       iOS prototype
  dosepair_android/   Android prototype
SKILL.md              Agent-skill entry point
```

## Backend API contract (v1)

- POST /api/analyze  body: { medications: [{name, dose?, route?}], disease? }
- GET  /health       returns { status, provider, version }
- Rate limit: 10 req/min per IP (express-rate-limit)
- Medications: array, 1-15 items, each with non-empty name (max 100 chars)
- Response JSON: { drugs[], interactions[], insights[], recommended_schedule[], summary }
- LLM provider selected via LLM_PROVIDER env var (anthropic|openai|grok|gemini|deepseek|qwen)
- API keys NEVER sent to client - proxy pattern only
- Stateless - no PHI stored

## Build Rules (NO EXCEPTIONS)

1. NO em dashes in code or comments - use regular hyphens only
2. Reference DosePair everywhere (never other names)
3. Import from components/ for all UI elements
4. All colors from tokens/ CSS variables
5. All fonts: Space Grotesk (headings), IBM Plex Sans (body), IBM Plex Mono (numbers/doses/times)
6. Disclaimer component on every results screen (always last)
7. Row limits: 15 desktop / 5 mobile
8. Backend never exposes any LLM API key to client
9. Version every new file (v1, v2, v3). Never overwrite.
